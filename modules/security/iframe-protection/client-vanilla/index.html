Understanding Iframe Protection

What is Clickjacking?

Clickjacking is an attack where a malicious site embeds your site in an iframe and overlays invisible elements to trick users into clicking on things they don't intend to.

Test 1: Protected Page (Should Fail)

✅ Protected Page

This page has X-Frame-Options: DENY and CSP frame-ancestors: 'none'

It should NOT load in this iframe.

// Server-side protection:
X-Frame-Options: DENY
Content-Security-Policy: frame-ancestors 'none'

Test 2: Vulnerable Page (Will Load)

❌ Vulnerable Page

⚠️ This page has NO iframe protection and WILL load.

This demonstrates why protection is necessary.

// ❌ NO PROTECTION - Vulnerable to clickjacking
// Missing X-Frame-Options header
// Missing CSP frame-ancestors directive

Protection Methods

1. X-Frame-Options Header

// ✅ DENY - No embedding allowed
X-Frame-Options: DENY

// ✅ SAMEORIGIN - Only same origin can embed
X-Frame-Options: SAMEORIGIN

// ⚠️ ALLOW-FROM (deprecated, use CSP instead)
X-Frame-Options: ALLOW-FROM https://example.com

2. Content Security Policy (CSP)

// ✅ No embedding allowed
Content-Security-Policy: frame-ancestors 'none'

// ✅ Same origin only
Content-Security-Policy: frame-ancestors 'self'

// ✅ Whitelist specific domains
Content-Security-Policy: frame-ancestors https://trusted.com

3. Iframe Sandbox Attribute

For iframes you control, use sandbox attribute:

// ✅ Secure iframe with sandbox
<iframe sandbox="allow-scripts allow-same-origin" src="..."></iframe>

// Sandbox restrictions:
// - No scripts (unless allow-scripts)
// - No forms (unless allow-forms)
// - No popups (unless allow-popups)
// - No same-origin access (unless allow-same-origin)

Best Practices

✅ DO's

  • Always set X-Frame-Options or CSP frame-ancestors
  • Use DENY or SAMEORIGIN for most sites
  • Use CSP frame-ancestors (more flexible than X-Frame-Options)
  • Use sandbox attribute for iframes you control
  • Test iframe embedding prevention regularly

❌ DON'Ts

  • Don't allow your site to be embedded without protection
  • Don't use X-Frame-Options: ALLOWALL (doesn't exist, but don't allow all)
  • Don't trust iframe content without sandbox
  • Don't allow iframes from untrusted domains
  • Don't rely on client-side protection only
